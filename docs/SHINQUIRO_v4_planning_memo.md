## 1. 概要

v3.5をベースに、**試験検索機能**を新設する。現行の長文検索に加え、英作文・リスニング・文法問題も検索可能にし、試験全体を俯瞰できる機能を提供する。

---

## 2. 新機能：試験検索（`/exam`）

### 2.1 画面構成

1つの画面内にタブ切替で2つのモードを持つ。

| タブ名 | 目的 |
|--------|------|
| **大問で探す** | 特定の問題タイプの大問をピンポイントで検索 |
| **試験で探す** | 条件に合う大問を含む試験全体を表示 |

### 2.2 「大問で探す」タブ

#### 検索フロー

1. **共通フィルター**: 試験区分（国公立/私立/共通テスト/資格）、年度（範囲指定）
2. **問題タイプを1つ選択**（複数選択不可）:
   - ライティング
   - リスニング
   - 文法
   - ※リーディングは選択肢に含めず、長文詳細検索（`/search`）へのリンクで誘導
3. **選択した問題タイプの固有フィルター**:
   - ライティング → 英作文タイプ（複数選択可）
   - リスニング → 解答形式（複数選択可）
   - 文法 → 問題タイプ（複数選択可）

#### 結果表示

条件に合う大問のみをフラットにリスト表示。

### 2.3 「試験で探す」タブ

#### 検索フロー

1. **共通フィルター**: 試験区分、年度
2. **4つのエリアを自由に設定**:

**リーディング**
- 合計単語数: 下限〜上限（デフォルトは下限なし〜上限なし）
- 日本語記述: あり / どちらでも / なし
- 英語記述: あり / どちらでも / なし

**ライティング**
- あり / どちらでも / なし（デフォルト: どちらでも）
- 「あり」選択時のみ → 英作文タイプの選択が可能（複数選択可）
- 「なし」選択時 → ライティングの大問を含む試験を除外

**リスニング**
- あり / どちらでも / なし（デフォルト: どちらでも）
- 「あり」選択時のみ → 解答形式の選択が可能（複数選択可）
- 「なし」選択時 → リスニングの大問を含む試験を除外

**文法問題**
- あり / どちらでも / なし（デフォルト: どちらでも）
- 「あり」選択時のみ → 問題タイプの選択が可能（複数選択可）
- 「なし」選択時 → 文法問題の大問を含む試験を除外

#### 結果表示

試験単位のカード表示。検索条件にヒットした大問は背景色やボーダーでハイライト表示。

#### 試験全体表示カードのイメージ

```
┌──────────────────────────────────────────────────────┐
│ [国公立] 東京大学 2025 前期 一般 全学部    制限時間:120分 │
│                                                      │
│  📖 第1問(長文) 850語 Lv.B | 📖 第2問(長文) 620語 Lv.C │
│  ✏️ 第3問(英作文) 自由英作文:100語以上                   │
│  🎧 第4問(リスニング) 選択問題|日本語記述               │
│  📝 第5問(文法) 空所選択補充                            │
│                                         [Amazon]      │
│              ▼ 詳細を見る                              │
└──────────────────────────────────────────────────────┘
```

※ヒットした大問はハイライト表示、それ以外は控えめに表示。

## 4. データ構造の変更

### 4.1 識別名の変更

アンダースコア区切りを導入し、`_` の前を試験ID、後を大問部分とする。

```
旧: tokyo2025kzen1A
新: tokyo2025kzen_1A
```

コード上で `識別名.split('_')` により試験IDと大問部分を分離可能。

### 4.2 大問IDのプレフィックス方式

全CSV横断でユニークにするため、問題タイプごとのアルファベットプレフィックスを付与。

| CSV | プレフィックス | 例 |
|-----|---------------|-----|
| reading.csv | R | R1, R2, R3... |
| writing.csv | W | W1, W2, W3... |
| listening.csv | L | L1, L2, L3... |
| grammar.csv | G | G1, G2, G3... |

各CSV内で通し番号を振り、プレフィックスで全体のユニーク性を保証。
既存の setsumon.csv・knowledge.csv・hashtags.csv・keywords.csv も大問IDを書き換える。

※注意点: ソート時は文字列比較になるため、プレフィックスを除去して数値比較する処理が必要。

### 4.3 CSV構成

#### 新設: exam.csv（試験マスタ）

```
試験ID, 試験区分, 大学名, 年度, 日程, 方式, 学部, 制限時間, ASIN
```

例: `tokyo2025kzen, 国公立, 東京大学, 2025, 前期, 一般, 全学部, 120, B0XXXXXXXX`

#### 変更: mondai.csv → reading.csv（長文問題）

試験情報を除去し、exam.csv から試験IDで参照する形に変更。

```
大問ID, 識別名, 大問番号, 本文語数, 本文レベル, ジャンル, 設問数, 文章記述, 出典
```

例: `R1, tokyo2025kzen_1A, 第1問, 850, B, 論説文, 8, あり, The Economist`

#### 新設: writing.csv（英作文問題）

```
大問ID, 識別名, 大問番号, 英作文タイプ
```

英作文タイプの値:
- 和文英訳
- 自由英作文:100語未満
- 自由英作文:100語以上

例: `W1, tokyo2025kzen_3, 第3問, 自由英作文:100語以上`

#### 新設: listening.csv（リスニング問題）

```
大問ID, 識別名, 大問番号, 解答形式
```

解答形式の値（パイプ区切りで複数指定可）:
- 選択問題
- 日本語記述
- 英語記述

例: `L1, tokyo2025kzen_4, 第4問, 選択問題|日本語記述`

#### 新設: grammar.csv（文法問題）

```
大問ID, 識別名, 大問番号, 問題タイプ
```

問題タイプの値（今後追加予定）:
- 空所選択補充
- 語句整序
- 誤文訂正
- 発音・アクセント

例: `G1, tokyo2025kzen_5, 第5問, 空所選択補充`

※将来的に小問単位のポイントもデータ化し検索可能にする予定。

#### 変更なし

以下のCSVは大問IDの書き換えのみで、構造は変更なし:

- `/exam` ページ（試験検索画面）

- 大学別一覧ページ: データ参照元の変更
